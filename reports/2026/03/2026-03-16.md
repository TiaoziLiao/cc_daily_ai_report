### ⭐⭐⭐⭐⭐ Trust Your Critic: Robust Reward Modeling and Reinforcement Learning for Faithful Image Editing and Generation

**Xiangyu Zhao, Peiyuan Zhang, Junming Lin, Tianhao Liang, Yuchen Duan, Shengyuan Ding, Changyao Tian, Yuhang Zang, Junchi Yan, Xue Yang** | [arXiv](https://arxiv.org/abs/2603.12247)
**领域**: #ImageGen #RL #RewardModel #T2I | **类型**: #method
**Upvotes**: 22 | **关联度**: #directly-relevant

> **TL;DR**: RL 已成为增强图像编辑和 T2I 生成的有前景范式，但当前 reward model 存在幻觉问题，给出噪声评分误导优化。本文提出 FIRM (Faithful Image Reward Modeling) 框架，通过定制化数据策展管线构建高质量评分数据集，训练 FIRM-Edit-8B 和 FIRM-Gen-8B 两个领域专用 reward model，并引入 Base-and-Bonus reward 策略平衡竞争目标，显著提升 RL 优化图像生成的可靠性。

**研究动机**:
- 现有 reward model（如 ImageReward, HPSv2）在充当 RL critic 时经常产生幻觉，对质量差异大的图像给出相近分数
- 噪声评分从根本上误导 RL 优化方向，导致生成模型学到虚假的奖励信号
- 图像编辑和 T2I 生成的评估标准不同（编辑需要保真度 + 指令遵循，生成需要质量 + 对齐），需要领域特定的 reward model

**核心方法**:
FIRM 框架包含三个核心组件：

1. **定制化数据策展管线**: 针对编辑和生成两个领域分别设计数据收集策略，构建包含细粒度质量标注的评分数据集，确保 reward model 学到准确的质量信号而非噪声。

2. **领域专用 Reward Model**: 训练两个 8B 参数的专用模型 — FIRM-Edit-8B 专注编辑场景的保真度和指令遵循度评估；FIRM-Gen-8B 专注生成场景的质量和文本-图像对齐评估。专用模型比通用 reward model 在各自领域表现显著更好。

3. **Base-and-Bonus Reward 策略**: 引入两个专门策略 — CME (Consistency-Modulated Execution) 用于编辑场景，确保编辑一致性；QMA (Quality-Modulated Alignment) 用于生成场景，在质量和对齐之间取得平衡。Base reward 保证基本质量，Bonus reward 鼓励超越基线。

**关键 Insight**:
1. **Reward Model 幻觉是 RL for Generation 的核心瓶颈** — 比算法选择（PPO vs DPO）更重要的是 reward signal 本身的质量，错误的 reward 会让模型越优化越差
2. **领域特定优于通用** — 编辑和生成的评估标准本质不同，一个通用 reward model 无法同时满足两者，分别训练专用模型是更好的策略
3. **Base-and-Bonus 解耦竞争目标** — 将基础质量保证和额外改进分开评估，避免单一 reward 信号在竞争目标间摇摆

**亮点与不足**:
- ✅ 系统性解决了 RL for image generation 中 reward model 不可靠的核心问题
- ✅ 提供了完整的数据策展→模型训练→RL 优化 pipeline
- ✅ FIRM-Edit 和 FIRM-Gen 分别在编辑和生成场景表现优异
- ❌ 8B 参数的 reward model 推理成本较高，在大规模 RL 训练中可能成为瓶颈
- ❌ 数据策展管线的构建成本较高，迁移到新领域需要重新设计

**对我研究的启发**: 这篇论文直接解决了 RL 优化生成质量时最核心的 reward model 质量问题。Base-and-Bonus 策略可以借鉴到任何需要平衡多个竞争目标的 RL for generation 场景（如 DiT 训练中平衡 FID 和 CLIP Score）。领域专用 reward model 的思路也值得在视频生成等场景探索。

### ⭐⭐⭐⭐⭐ Cheers: Decoupling Patch Details from Semantic Representations Enables Unified Multimodal Comprehension and Generation

**Authors** | [arXiv](https://arxiv.org/abs/2603.12793)
**领域**: #Diffusion #Multimodal #ImageGen | **类型**: #method
**Upvotes**: 14 | **关联度**: #directly-relevant

> **TL;DR**: 统一多模态理解和生成一直是前沿热点，但两个任务需要不同的解码机制和视觉表征，联合优化存在根本矛盾。Cheers 提出将 patch 级别的细节从语义表征中解耦，在不牺牲理解能力的前提下提升生成保真度，首次实现理解-生成真正统一的多模态模型。

**研究动机**:
- 理解任务需要高层语义表征（"这是什么"），生成任务需要低层细节表征（"像素怎么画"）
- 现有统一模型要么牺牲理解准确度换取生成质量，要么相反
- 两种表征在共享特征空间中相互干扰，是一个根本性的架构问题

**核心方法**:
Cheers 的核心创新在于表征解耦：将 patch-level 的视觉细节与语义表征分离到不同的子空间，语义分支负责多模态理解的稳定性，细节分支负责图像生成的保真度。两个分支共享底层特征提取，但在高层各自独立。这一设计使得理解和生成可以独立优化而不相互干扰。

生成侧采用 diffusion 解码器，接收细节分支的特征作为条件；理解侧使用语义分支的特征进行 token 预测。两个分支通过轻量级交互模块保持信息流通。

**关键结果**: 在理解和生成两个方向均达到 SOTA 或 competitive 水平，相比之前的统一模型（如 Janus, SEED）在两个方向的综合表现提升显著。

**关键 Insight**:
1. 统一多模态模型的关键不在于用同一个表征做所有事，而在于**优雅地解耦**不同粒度的信息
2. 解耦后的轻量级交互模块证明，理解和生成之间确实存在可共享的底层信息

**对我研究的启发**: 表征解耦的思路可直接应用于 DiT 架构中分离语义控制和细节生成。尤其在可控生成场景，将控制信号（文本/布局）和生成细节分离可能带来更好的控制精度。

### ⭐⭐⭐⭐⭐ From Sparse to Dense: Multi-View GRPO for Flow Models via Augmented Condition Space

[arXiv](https://arxiv.org/abs/2603.12648)
**领域**: #FlowMatching #T2I #RL #GRPO | **类型**: #method
**Upvotes**: (快速索引) | **关联度**: #directly-relevant

> **TL;DR**: 将 GRPO (Group Relative Policy Optimization) 从 LLM 领域迁移到 Flow Matching 模型，通过增广条件空间实现多视角 GRPO 训练，解决了 flow model RL 优化中的稀疏 reward 问题。这是 GRPO + Flow Matching 的首个系统性工作。

**研究动机**:
- GRPO 在 LLM 中展现出强大的 RL 优化效果（如 DeepSeek-R1），但在视觉生成领域的应用还处于早期
- Flow Matching 模型的连续时间特性使得标准 RL 算法难以直接应用
- 图像生成的 reward signal 天然稀疏（只有最终图像有评分），导致训练信号不足

**核心方法**:
核心思路是将 GRPO 的 group relative 比较机制从 token 序列空间扩展到 flow model 的连续轨迹空间。通过增广条件空间（Augmented Condition Space），在同一 prompt 下生成多个不同视角/条件的样本，形成更丰富的 group 进行 relative 比较。这解决了稀疏 reward 问题 — 同一 group 内的多视角样本提供了细粒度的比较信号。

**关键结果**: 在 T2I 生成质量和文本对齐度上显著优于 PPO 和 DPO baseline，且训练更稳定。

**关键 Insight**:
1. **GRPO→Flow Model 的迁移关键在于构造有意义的 group** — 增广条件空间是解决方案
2. **多视角采样提供了 reward densification** — 这是一个通用的技巧，可用于任何 sparse reward 的生成 RL 场景
3. **Flow Matching 的连续性使得 GRPO 的优势更明显** — 相比 diffusion model，flow model 的直线轨迹更适合 trajectory-level 的优化

**对我研究的启发**: GRPO 在 flow model 上的成功验证了 LLM RL 技术向视觉生成迁移的可行性。增广条件空间的 reward densification 技巧可直接用于 DiT + RL 训练。这是一个值得立即跟进的方向。

### ⭐⭐⭐⭐ EVATok: Adaptive Length Video Tokenization for Efficient Visual Autoregressive Generation

**Tianwei Xiong, Jun Hao Liew, Zilong Huang** | [arXiv](https://arxiv.org/abs/2603.12267)
**领域**: #VideoGen #Tokenizer #Autoregressive | **关联度**: #directly-relevant

> **TL;DR**: 传统视频 tokenizer 对所有帧均匀压缩，忽略内容动态变化。EVATok 引入自适应长度机制，根据内容复杂度动态分配 token 数量 — 高运动帧多分配，静态帧少分配，在重建质量和生成计算成本间实现更优平衡。

**核心方法**: EVATok 在 tokenizer 中引入动态决策模块，评估每个 temporal block 的运动复杂度和视觉信息量，自适应调整该 block 的 token 序列长度。高运动/高信息量区域获得更多 token 保留细节，静态/重复区域用更少 token 表示。

**关键结果**: 在相同总 token 数下重建质量优于固定长度 tokenizer；在相同重建质量下减少约 30-40% 的 token 数量。

**Insight / 启发**:
